# Roadmap

The course is built incrementally. Add full module content only when that topic begins.

## Phase 0: Foundation

- Project setup
- Agent handoff context
- Review rubric
- Module template
- First TypeScript path skeleton

## Phase 1: TypeScript for LLD

- Classes and interfaces
- Access modifiers and encapsulation
- Abstract classes vs interfaces
- Composition vs inheritance
- Enums, literal types, and discriminated unions
- Generics for reusable design
- Error handling and result-style flows
- Modules and dependency boundaries
- Unit testing design behavior

## Phase 2: Object Modeling

- Entities
- Value objects
- Domain services
- Repositories and in-memory stores
- State transitions
- Invariants
- API/method design

## Phase 3: Design Principles

- SOLID
- DRY
- KISS
- YAGNI
- Separation of concerns
- Encapsulation
- Composition over inheritance
- Law of Demeter
- Tell, don't ask
- High cohesion and low coupling

## Phase 4: Design Smells

- God class
- Long parameter list
- Primitive obsession
- Feature envy
- Shotgun surgery
- Tight coupling
- Anemic domain model
- Boolean flag arguments
- Inheritance abuse

## Phase 5: Design Patterns

- Creational: Factory, Abstract Factory, Builder, Prototype, Singleton with caution
- Structural: Adapter, Facade, Decorator, Composite, Proxy
- Behavioral: Strategy, Observer, State, Command, Chain of Responsibility, Template Method, Iterator

## Phase 6: Classic LLD Problems

- Parking Lot
- Splitwise
- Elevator System
- Logging Framework
- Rate Limiter
- BookMyShow or ticket booking
- Vending Machine
- Chess
- Snake and Ladder

## Phase 7: Product Startup LLD Problems

- Feature flags
- RBAC and permissions
- Team invites
- Subscription billing
- Notification preferences
- Audit logs
- Workflow automation
- Form builder
- Coupon engine
- Multi-tenant workspace
- Webhook delivery system

## Phase 8: Capstones and Mock Interviews

- Combine multiple patterns and principles.
- Practice requirements clarification.
- Present class/API design.
- Defend tradeoffs and extension points.
- Refactor under changing requirements.
